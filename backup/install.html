安装前的准备

注册一个Github账号（如有则可以跳过）

安装Git：

  • Mac OS X 下载地址
  • Windows 下载地址
  • Linux 下载地址（可能被墙）

创建项目

首先，创建你的Blog仓库，点击创建

注意！以下代码中的username都要替换成你自己的github的账号名称

仓库名称（Repository name）为username(请确保跟你的账号名相同).github.com

打开终端，输入：（注意username应替换成你自己的账号名称）

$ mkdir username.github.com
$ cd username.github.com
$ git init
$ git checkout --orphan master

其次，在该文件夹内放入源码

初始化仓库，提交并push到Github：（注意username应替换成你自己的账号名称）

$ git add .
$ git commit -m "first post"
$ git remote add origin https://github.com/username/username.github.com.git
$ git push origin master

稍等3~10分钟，打开username.github.com就可以看到刚才新建的页面了，就是这么简单。当然你也可以为你的Blog仓库绑定独立域名，具体做法我们稍后提到。

经典错误

错误1

! [rejected]        master -> master (non-fast-forward)
error: failed to push some refs to 'https://github.com/username/username.github.com.git'

解决方法：

$ git push -f

错误2

fatal: https://github.com/username/username.git/info/refs not found:
did you run git update-server-info on the server?

解决方法：

$ git remote -v

确认连接是否为你git的连接，如果不是：

$ git remote set-url origin https://github.com/username/username.git

(username注意是区分大小写的)

如果同步失败，也可以选择使用github程序同步：Windows Mac

设置

在Github上打开仓库内的_config.yml并点击编辑模式（也可用编辑器打开_config.yml）。

/* 以下内容可自由填写，冒号后有空格 */
url: //博客的域名，如没有域名，则为username.github.com
title: //博客的名字
author: //博客的作者
keywords: //博客的关键词，为了让搜索引擎检索到你的博客的关键词，多个关键词用英文逗号,隔开（逗号前后不需空格）
description: //博客的简介，为了让搜索引擎检索到你的博客的简介，简介在60字以内即可
email: //博主的邮箱
googleAnalyticsID: //谷歌分析ID
disqusShortname: //disqus短码（如果没有，去disqus注册申请，否则会影响网站速度）
