如何修改域名？

修改仓库内的CNAME文本文件，里面写入你要绑定的域名，比如example.com或者xxx.example.com

一级域名：DNS新建一条A记录，指向204.232.175.78

二级域名：DNS新建一条CNAME记录，指向username.github.com（请将username换成你的用户名）

如何修改博客网页小图标？

网页小图标是浏览器自动识别favicon.ico图标从而显示在浏览器上的，没有favicon.ico的可以在favicon.cc制作，完成后上传到仓库根目录并覆盖掉原来的即可。

如何撰写日志？

Jekyll博客的日志都必须使用Markdown撰写，所以你需要一个Markdown编辑软件：

  • Prose，网页版Markdown，专供Jekyll博客撰写Markdown
  • MaHua，中文网页版Markdown
  • Dillinger，网页版Markdown
  • MarkdownPad，适用于Windows，需要安装Microsoft .NET Framework 4.0
  • Mou，适用于Mac OS X
  • ReText，适用于Linux
  • Vim，适用于所有系统，新手不建议使用

什么是Markdown？

Markdown 是一种轻量级标记语言，创始人为John Gruber和Aaron Swartz。它允许人们“使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML(或者HTML)文档”。这种语言吸收了很多在电子邮件中已有的纯文本标记的特性。

来自维基百科

Jekyll支援所有Markdown语法吗？

Jekyll支持最基础的Markdown语法：

标题:

# 标题 大
                  
## 标题 中
                  
### 标题 小

强调:

*该文本会被强调*
                  
**该文本会加粗**

链接：

[链接注释](网址)

图片：

![图片注释](图片网址)

引用

> 引用文本

无顺序列表

* 文本 1
* 文本 2
  * 文本 2a（前面2个空格）
  * 文本 2b（前面2个空格）

顺序列表

1. 文本 1
2. 文本 2

代码

    代码文本 1（前面空4格）
    代码文本 2（前面空4格）
    代码文本 3（前面空4格）

需要注意的是，行末加两个或多个空格才是真正的换行，否则正常的一个回车就像在HTML代码中一样，被当作空格处理。

所以，这样的Markdown书写在诗歌换行的情况下是错误的：

# 再别康桥
轻轻的我走了，
正如我轻轻的来；
我轻轻的招手，
作别西天的云彩。

正确的写法是：

# 再别康桥
                  
轻轻的我走了，
                  
正如我轻轻的来；
                  
我轻轻的招手，
                  
作别西天的云彩。

如何更新Plumbious？（同样适用于更新主题，提交博文）

将_include，_post，assets里面的文件以及config.yml，CNMAE设置文件做好备份，将Plumbious直接覆盖至仓库的根目录，然后恢复备份

打开终端：

$ cd username.github.com
$ git commit -m "update"
$ git remote add origin git@github.com:username/username.github.com.git
$ git push origin master
